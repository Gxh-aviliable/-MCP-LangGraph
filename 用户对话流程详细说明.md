# 用户对话流程详细说明

## 1. 整体架构概览

系统包含两个核心流程图：
- **chat_graph**: 对话流程图，用于收集用户需求
- **plan_graph**: 规划流程图，用于生成行程（有两种模式）

两种规划模式：
- **smart 模式**: 智能规划图（高效流水线，并行执行）
- **react 模式**: ReAct Agent 图（动态推理，循环执行）

---

## 2. API 入口流程 (main.py)

```
用户请求 → /api/chat 或 /api/chat/stream
    │
    ├─ 有 session_id?
    │   ├─ Yes → 从 session_manager 获取已有会话
    │   └─ No  → 创建新会话 (ChatSession)
    │           └─ await session.initialize(agent_mode)
    │
    ├─ 首次访问且无消息? (not session_id and not message)
    │   ├─ Yes → await session.start()
    │   │          └─ 返回问候消息
    │   └─ No  → await session.chat(user_message)
    │              └─ 处理用户消息
    │
    └─ 返回响应 (SSE 流式或 JSON)
```

---

## 3. ChatSession 对话流程 (chat_graph)

### 3.1 chat_graph 结构

```python
# trip_agent.py:332-354
workflow.set_entry_point("analyzer")

analyzer → response → END
```

这是一个简单的两步流程：
- **analyzer**: 分析用户消息，提取旅行信息
- **response**: 根据分析结果生成回复

### 3.2 conversation_stage 阶段流转

```
┌─────────────────────────────────────────────────────────────────┐
│                     对话阶段流转图                               │
└─────────────────────────────────────────────────────────────────┘

greeting (问候)
    │
    │ 用户发送消息
    ↓
collecting (收集必要参数)
    │
    │ 必要参数完成 (ready_to_plan = True)
    │   必要参数: origin, city, start_date, end_date
    ↓
optional_collecting (可选参数引导)
    │
    │ 用户跳过 或 达到3轮上限 或 可选参数填满
    ↓
confirming (确认生成)
    │
    │ 用户确认 (回复"是"/"确认"/"生成")
    ↓
planning (正在规划)
    │
    │ 规划完成
    ↓
refining (调整行程)
    │
    │ 用户确认满意 (回复"确认"/"满意")
    ↓
done (完成)
```

## 4. 各阶段详细说明

### 4.1 greeting 阶段

**触发条件**: 用户首次访问，无 session_id 且无 message

**调用方法**: `session.start()`

**节点调用**:
- 不调用 chat_graph 节点
- 直接返回预设的 GREETING_MESSAGE

**状态更新**:
```python
initial_state = create_initial_state()
await chat_graph.aupdate_state(config, initial_state)
```

**返回内容**:
- reply: GREETING_MESSAGE ("您好！我是旅行规划助手...")
- stage: "greeting"
- collected_info: {}
- missing_fields: ['origin', 'city', 'start_date', 'end_date']

---

### 4.2 collecting 阶段

**触发条件**: 用户发送消息，necessary 参数未收集完成

**调用方法**: `session.chat(user_message)`

**节点调用顺序**:
```
chat_graph.ainvoke(analyze_input, config)
    │
    ├─ analyzer (requirement_analyzer_node)
    │   ├─ 提取用户消息中的旅行信息
    │   ├─ 合并到 collected_info
    │   ├─ 更新 missing_fields
    │   ├─ 调用 extract_special_instructions (识别特殊说明)
    │   └─ 判断 ready_to_plan
    │
    └─ response (response_generator_node)
        └─ 根据阶段生成回复文本
```

**状态更新**:
```python
{
    "collected_info": new_collected,       # 合后的信息
    "missing_fields": missing,             # 缺失字段
    "ready_to_plan": ready,                # 是否就绪
    "conversation_stage": "optional_collecting" if ready else "collecting",
    "special_instructions": {...},         # 用户特殊说明
}
```

---

### 4.3 optional_collecting 阶段

**触发条件**: necessary 参数已完成，optional 参数未收集

**调用方法**: `session.chat(user_message)` → `_handle_optional_collection()`

**节点调用**:
```
optional_guidance_node(llm, optional_input)
    │
    ├─ 检查用户是否跳过 ("直接生成"/"不用了")
    ├─ 提取可选参数 (interests, budget, accommodation, transport)
    ├─ 判断是否结束引导
    │   ├─ 用户明确跳过
    │   ├─ 达到 MAX_OPTIONAL_ROUNDS (3轮)
    │   └─ 所有可选参数已填写
    │
    └─ 更新状态
        ├─ optional_collected: True/False
        ├─ optional_skipped: True/False
        ├─ optional_round: 轮次计数
        └─ conversation_stage: "confirming" 或 "optional_collecting"
```

**可选参数列表**:
- interests: 兴趣偏好
- budget_per_day: 每日预算
- accommodation_type: 住宿类型
- transportation_mode: 交通偏好

---

### 4.4 confirming 阶段

**触发条件**: 可选参数收集完成

**调用方法**: `session.chat(user_message)`

**处理逻辑**:
```python
if stage == "confirming" and ready_to_plan:
    # 1. 使用 LLM 分析用户消息（复用 optional_guidance_node，统一识别方式）
    result = await optional_guidance_node(self.llm, {
        **current_values,
        "user_feedback": user_message,
    })

    # 2. 检查是否提取到新的可选参数
    new_collected = result.get('collected_info', {})
    if new_collected != collected_info:
        # 用户提供了可选参数，更新状态并继续确认
        return {"reply": "已记录您的信息，现在为您生成行程吗？", ...}

    # 3. 检查是否确认生成
    confirm_result = await confirm_check_node({"user_feedback": user_message})
    if confirm_result == "confirmed":
        return await self._generate_and_return(collected_info, current_values)

    # 4. 未确认，保持当前状态
    return {"reply": "请确认是否生成行程...", "stage": "confirming"}
```

**特点**:
- 复用 `optional_guidance_node` 进行可选参数识别，与 `optional_collecting` 阶段保持一致
- 用户可以用自然语言表达偏好（如"我喜欢安静的地方"），LLM 能理解并提取
- 确认关键词: ['是', '好', '生成', '可以', '确认', '没问题', 'ok', 'yes', '开始']

---

### 4.5 planning 阶段

**触发条件**: 用户确认生成

**调用方法**: `_generate_and_return(collected_info, current_values)`

**节点调用** (plan_graph):
- 根据模式选择不同的图结构

详见第 5 章"规划流程图详解"

---

### 4.6 refining 阶段

**触发条件**: 行程已生成

**调用方法**: `session.chat(user_message)` → `_handle_plan_adjustment()`

**处理逻辑**:
```python
# 1. 检查是否确认满意
if user_message.strip() in CONFIRM_KEYWORDS:
    return {"stage": "done", ...}

# 2. 根据模式处理调整
if agent_mode == "react":
    return _handle_react_feedback(...)
else:
    return _handle_smart_feedback(...)
```

**smart 模式调整流程**:
```
_handle_smart_feedback()
    │
    ├─ parse_intent_node(llm, intent_state)
    │   └─ 解析意图: modify_attractions/modify_hotels/modify_schedule
    │
    └─ adjust_plan_node(llm, adjust_state)
        └─ 根据意图修改行程 JSON
```

**react 模式调整流程**:
```
_handle_react_feedback()
    │
    └─ plan_graph.ainvoke(plan_input)
        └─ 重新进入 ReAct 推理循环
```

---

### 4.7 done 阶段

**触发条件**: 用户确认满意行程

**返回内容**:
- reply: "感谢您的确认！祝您旅途愉快！"
- stage: "done"
- plan: 最终行程

## 5. 规划流程图详解 (plan_graph)

### 5.1 Smart 模式 - 智能规划图

- LangGraph 的直接边会自动并行执行
- 一个节点有多条出边时，所有目标节点并行启动
- 汇聚节点自动等待所有并行分支完成

### 5.2 React 模式 - ReAct Agent 图

```
┌─────────────────────────────────────────────────────────────────┐
│                    ReAct 模式规划图                              │
│              (create_react_agent_graph)                          │
└─────────────────────────────────────────────────────────────────┘

Entry Point
    │
    ↓
┌──────────────┐
│  reasoning   │  推理节点
│              │  ├─ 分析当前状态
│              │  ├─ 决定下一步行动
│              │  ├─ 输出: {
│              │       thought: "思考内容",
│              │       action: "attraction_query|plan_trip|...",
│              │       action_reason: "选择原因",
│              │       confidence: 0.7 }
└──────┬───────┘
       │
       ↓
┌──────────────┐
│   action     │  行动节点

│              │  ├─ 可能调用: 景点/天气/酒店/交通工具
│              │  ├─ 可能调用: plan_trip_node
│              │  ├─ 可能调用: adjust_plan_node
│              │  └─ 输出: 工具调用结果或行程
└──────┬───────┘
       │
       ↓
┌──────────────┐

│              │  ├─ 解析 action 结果
│              │  ├─ 提取结构化数据
│              │  └─ 输出: {
│              │       observation: "观察到什么",
│              │       data: {...} }
└──────┬───────┘
       │
       ↓
┌──────────────┐
│  reflection  │  反思节点
│              │  ├─ 评估当前进度

│              │  └─ 输出: {
│              │       reflection: "反思内容",

│              │       quality_score: 0.6,

└──────┬───────┘
       │
       │ 条件路由 (should_continue)
       ↓
    ┌────────────┐
    │should_     │
    │continue?   │
    └────────────┘
      /        \
  True          False
  (或 iter<10)  (或 quality>=0.8)
    /            \
   ↓              ↓
┌──────────┐    ┌────────┐
│reasoning │    │  END   │
│ (循环)   │    │        │
└──────────┘    │返回    │
    ↑           │final_  │
    │           │plan    │
    └───────────┴────────┘

终止条件:
1. next_action == 'finish'
2. quality_score >= 0.8
3. iteration_count >= 10 (最大保护)
4. should_continue == False 且有 final_plan
```

## 6. 各节点功能详解

### 6.1 对话节点 (chat_graph)

## 7. 状态字段说明 (ChatAgentState)

### 7.1 必要参数字段
```python
origin: str           # 出发城市
city: str             # 目的地城市
start_date: str       # 开始日期 YYYY-MM-DD
```

## 8. SSE 流式输出事件

```
event: session   → 发送会话 ID
event: message   → 发送回复内容
event: stage     → 发送阶段状态 (stage, collected_info, missing_fields)
event: plan      → 发送行程数据
event: done      → 发送完成信号
event: error     → 发送错误信息
```

## 9. 流程总结图

```
┌───────────────────────────────────────────────────────────────────────────┐
│                           完整用户对话流程                                  │
└───────────────────────────────────────────────────────────────────────────┘

用户请求
    │
    ├─ 首次访问 (无 session_id, 无 message)
    │   │
    │   └─ session.start()
    │       └─ 返回问候消息 (greeting 阶段)
    │
    └─ 有消息
        │
        └─ session.chat(user_message)
            │
            ├─ greeting → chat_graph.ainvoke
            │   └─ analyzer → response → collecting
            │
            ├─ collecting → chat_graph.ainvoke
            │   └─ analyzer → response → collecting/optional_collecting
            │
            ├─ optional_collecting → optional_guidance_node
            │   └─ 多轮引导 → confirming
            │
            ├─ confirming → confirm_check_node
            │   ├─ 确认 → plan_graph.ainvoke (planning)
            │   └─ 未确认 → 保持 confirming
            │
            ├─ planning → plan_graph.ainvoke
            │   ├─ Smart: tool_selector → 并行 → hotel → plan → reflection
            │   └─ ReAct: reasoning → action → observation → reflection (循环)
            │   └─ 输出 final_plan → refining
            │
            ├─ refining → _handle_plan_adjustment
            │   ├─ Smart: parse_intent → adjust_plan → refining
            │   └─ ReAct: 重新进入推理循环 → refining
            │   └─ 确认满意 → done
            │
            └─ done → 会话结束
```
