# AI 智能旅行规划助手 - 简历项目经验

**时间**: 2025-12 ~ 2026-01

**技术栈**: Python、FastAPI、Vue3、LangGraph、LangChain、WebSocket、Docker

---

## 📋 项目描述

基于多智能体架构的全栈 AI 旅行规划系统，集成了景点搜索、天气查询、酒店推荐、行程规划等功能。通过 LangGraph 编排多个 AI 智能体，调用高德地图、天气 API 等外部服务，为用户生成个性化的旅行计划。采用前后端分离架构（FastAPI + Vue3），支持实时进度反馈和 Docker 容器化部署。

---

## 🎯 核心工作与成就

### **后端开发 (FastAPI + LangGraph 多智能体)**

• **多智能体系统设计**: 使用 LangGraph 编排景点专家、天气专家、酒店专家、规划专家四个智能体，通过 MCP (Model Context Protocol) 调用高德地图、天气查询等外部工具，实现了智能体间的协作与数据流转。

• **异步 API 接口开发**: 基于 FastAPI 构建了 7 个 RESTful 接口与 WebSocket 长连接，支持同步规划接口 (`/api/plan`) 和实时进度反馈接口 (`/ws/plan`)，使用异步编程实现高并发请求处理，单个请求超时时间设置为 5 分钟。

• **数据模型设计**: 使用 Pydantic 定义了完整的请求/响应数据模型，包括 TripRequest、TripPlan、DayPlan、Attraction、Hotel、WeatherInfo 等 10+ 个数据类，确保了数据的类型安全和自动验证。

• **实时进度管理**: 实现了基于 WebSocket 的双向通信机制，前端通过长连接接收后端的实时进度消息（"正在搜索景点"、"正在查询天气" 等），完整显示了多智能体的执行过程，提升了用户体验。

• **跨域资源共享 (CORS)**: 配置了 FastAPI 的 CORS 中间件，使用 `allow_origins=["*"]` 支持前端跨域请求，同时在生产环境中可灵活限制具体域名。

### **前端开发 (Vue3 + Element Plus)**

• **响应式界面设计**: 采用现代化 UI 设计（渐变背景、卡片布局、Flexbox 弹性布局），使用 Element Plus 组件库构建了表单、日期选择、下拉列表、结果卡片等功能模块，支持桌面、平板、手机等多设备自适应。

• **状态管理架构**: 基于 Pinia 实现了全局状态管理，定义了 tripRequest、tripPlan、isLoading、error、progressMessages 等状态，通过 computed 计算属性实现了动态数据转换（如旅行天数计算、表单有效性验证）。

• **表单交互与验证**: 实现了实时表单验证（城市选择、日期范围、兴趣爱好、预算等字段），计算显示旅行天数并给出超长旅行提示，支持添加/删除兴趣标签，提供了友好的用户输入体验。

• **异步请求处理**: 使用 Axios 封装了 HTTP 客户端，配置了 300 秒请求超时，实现了响应拦截器处理错误情况（如 503 服务不可用），在请求期间显示加载对话框和进度消息。

• **结果展示与操作**: 设计了多卡片结构展示旅行计划（基本信息、预算分析、天气预报、每日行程），支持 JSON 导出和打印功能，添加了天气图标、餐饮类型图标等视觉反馈。

### **DevOps 与部署**

• **容器化部署**: 编写了后端和前端的 Dockerfile，其中前端采用了多阶段构建（Node 18-Alpine 编译 → Nginx Alpine 部署），实现了镜像大小最优化（前端镜像 < 50MB）。

• **容器编排**: 配置了 docker-compose.yml，定义了后端服务、前端服务、网络、环境变量等，实现了一键启动整个应用栈，包括服务健康检查、依赖关系管理、卷挂载等。

• **启动脚本开发**: 为 Windows、Linux/Mac 分别编写了交互式启动脚本（start.bat、start.sh），提供了 7 个菜单选项（安装依赖、启动服务、清理文件等），降低了项目启动的复杂度。

• **Nginx 反向代理**: 配置了 Nginx 反向代理（前端路由支持、API 代理、WebSocket 代理、静态资源缓存），实现了前后端的无缝集成，在生产环境支持长连接超时 86400s。

### **测试与文档**

• **完整测试套件**: 编写了 test_api.py 包含健康检查、城市查询、兴趣分类、住宿类型、同步规划、WebSocket 实时规划等 6+ 个测试用例，支持一键运行所有测试并生成报告。

• **详细技术文档**: 提供了 QUICKSTART.md（快速启动指南）、README.md（完整文档）、IMPLEMENTATION_SUMMARY.md（实现总结）、FILE_MANIFEST.md（文件清单）等 4 份文档，共 2000+ 行内容，覆盖了系统架构、部署步骤、API 接口、故障排除等全方位内容。
