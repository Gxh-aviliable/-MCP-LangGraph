## 二、核心组件说明

### 2.1 两个 LangGraph 流程图

| 流程图 | 文件位置 | 功能 | 节点 |
|--------|----------|------|------|
| `chat_graph` | `trip_agent.py:115-137` | 对话收集需求 | analyzer → response |
| `plan_graph` | `trip_agent.py:139-170` | 规划生成行程 | 景点 → 天气 → 酒店 → 规划器 |

### 2.2 状态定义 (`ChatAgentState`)

```python
class ChatAgentState(TypedDict):
    # 用户输入信息
    city: str                    # 目标城市
    start_date: str              # 开始日期
    end_date: str                # 结束日期
    interests: List[str]         # 兴趣偏好

    # 中间结果 (累加)
    attractions_data: List[Dict] # 景点数据
    weather_data: List[Dict]     # 天气数据
    hotels_data: List[Dict]      # 酒店数据

    # 对话相关
    messages: List[Dict]         # 对话历史
    user_feedback: str           # 用户反馈
    conversation_stage: str      # 对话阶段
    collected_info: Dict         # 已收集信息
    missing_fields: List[str]    # 缺失字段
    ready_to_plan: bool          # 是否可规划

    # 最终结果
    final_plan: Dict             # 最终行程
    bot_reply: str               # 机器人回复
```

## 六、对话阶段说明

| 阶段 | 说明 | 用户操作 |
|------|------|----------|
| `greeting` | 问候阶段 | 系统发送欢迎消息 |
| `collecting` | 收集信息阶段 | 用户描述需求，系统追问缺失字段 |
| `confirming` | 确认阶段 | 系统展示收集的信息，询问是否生成 |
| `planning` | 规划阶段 | 执行 plan_graph，生成行程 |
| `refining` | 调整阶段 | 用户查看行程，可提出修改建议 |
| `done` | 完成阶段 | 用户确认满意，会话结束 |

## 八、总结

核心是 **LangGraph 的两个流程图**：

1. **chat_graph**: 对话收集需求
   - `analyzer_node`: 解析用户消息，提取旅行信息
   - `response_node`: 根据阶段生成回复

2. **plan_graph**: 规划生成行程
   - `attraction_expert`: 搜索景点
   - `weather_expert`: 查询天气
   - `hotel_expert`: 搜索酒店
   - `plan_trip`: LLM 生成结构化行程

数据流转:
```
用户输入 → analyzer_node → collected_info →
确认 → plan_graph → attractions/weather/hotels_data →
plan_trip → final_plan → 返回用户
```
